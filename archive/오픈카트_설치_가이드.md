# 설치 가이드 수정본

## 구성

* 쇼핑몰: **OpenCart 4.1.0.3**
* 웹서버: **Apache 2**
* DB: **MySQL**
* OS: **Ubuntu 22.04**

---

# 1) Ubuntu 22.04에서 OpenCart 설치 절차

## 1단계. 서버 패키지 업데이트

새로 준비한 Ubuntu 서버에서는 먼저 패키지 목록과 설치된 패키지를 최신 상태로 맞춥니다.

```bash
sudo apt update
sudo apt upgrade -y
```

---

## 2단계. Apache, PHP, MySQL 설치

OpenCart 4.x를 구동하기 위해 Apache, MySQL, PHP와 필요한 확장 모듈을 함께 설치합니다.

```bash
sudo apt install -y apache2 mysql-server \
php libapache2-mod-php php-mysql \
php-curl php-gd php-mbstring php-xml php-zip php-cli \
unzip wget
```

설치 후 서비스 상태를 확인합니다.

```bash
sudo systemctl enable apache2 mysql
sudo systemctl start apache2 mysql
sudo systemctl status apache2
sudo systemctl status mysql
```

Apache와 PHP 연동 반영을 위해 Apache를 한 번 재시작합니다.

```bash
sudo systemctl restart apache2
```

---

## 3단계. OpenCart용 DB 생성

MySQL에 접속합니다.

```bash
sudo mysql -u root
```

MySQL 프롬프트 안에서 아래를 실행합니다.

```sql
CREATE DATABASE opencart DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'opencartuser'@'localhost' IDENTIFIED BY 'strongpass!1234';
GRANT ALL PRIVILEGES ON opencart.* TO 'opencartuser'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

예시에서 바꿔도 되는 값은 다음과 같습니다.

* DB 이름: `opencart`
* 사용자: `opencartuser`
* 비밀번호: 원하는 강한 비밀번호

추후 설치 화면에서는 여기서 만든 **DB 이름, DB 사용자명, DB 비밀번호**를 그대로 입력합니다. 이 값은 리눅스 로그인 계정이나 `sudo` 비밀번호와는 다릅니다.

필요하면 현재 생성된 사용자와 데이터베이스를 아래처럼 확인할 수 있습니다.

```bash
sudo mysql -e "select user, host from mysql.user"
sudo mysql -e "show databases"
```

## 4단계. OpenCart 소스 받기

본 가이드에서는 **VM 안에서 직접 OpenCart 4.1.0.3 ZIP 파일을 다운로드한 뒤 설치하는 방식**으로 진행합니다.

OpenCart는 압축 해제 후 생성되는 `upload` 폴더 자체를 웹 루트로 쓰는 것이 아니라, **`upload` 폴더 안의 내용만 실제 웹 루트에 배치**해야 합니다.

먼저 `~`에서 ZIP 파일을 내려받습니다.

```bash
cd ~
wget https://github.com/opencart/opencart/releases/download/4.1.0.3/opencart-4.1.0.3.zip
```

그다음 웹 루트 디렉터리를 만들고, 해당 위치에서 압축을 해제합니다.

```bash
sudo mkdir -p /var/www/opencart
cd /var/www/opencart
sudo unzip ~/opencart-4.1.0.3.zip
```

압축 해제 후 생성된 `upload` 폴더 안의 파일들만 현재 경로(`/var/www/opencart`)로 복사합니다.

```bash
sudo cp -r upload/. /var/www/opencart/
sudo rm -rf upload
sudo rm -f ~/opencart-4.1.0.3.zip
```

## 5단계. 설정 파일 준비 및 권한 설정

OpenCart 설치를 위해 설정 파일을 준비합니다.

```bash
sudo cp /var/www/opencart/config-dist.php /var/www/opencart/config.php
sudo cp /var/www/opencart/admin/config-dist.php /var/www/opencart/admin/config.php
```

복사 후에는 두 파일이 실제로 생성되었는지 바로 확인합니다. 이 확인 단계를 넣어 두면 나중에 설치 화면의 사전 요구사항 검사에서 `/var/www/opencart/config.php` 또는 `/var/www/opencart/admin/config.php`가 `Missing`으로 표시되는 문제를 미리 잡을 수 있습니다.

```bash
ls -l /var/www/opencart/config.php /var/www/opencart/admin/config.php
```

정상이라면 두 파일이 모두 보여야 합니다. 만약 `admin/config.php`가 없으면 아래 명령으로 다시 생성합니다.

```bash
sudo cp /var/www/opencart/admin/config-dist.php /var/www/opencart/admin/config.php
```

또한 SEO URL 기능을 사용할 예정이라면 `htaccess.txt`를 `.htaccess`로 변경해야 합니다. OpenCart 공식 문서는 SEO URL 사용 전 이 설정을 요구합니다. [OpenCart][7]

```bash
sudo mv /var/www/opencart/.htaccess.txt /var/www/opencart/.htaccess
```

권한은 아래처럼 설정합니다.

```bash
sudo chown -R www-data:www-data /var/www/opencart
sudo find /var/www/opencart -type d -exec chmod 755 {} \;
sudo find /var/www/opencart -type f -exec chmod 644 {} \;
```

## 6단계. Apache 사이트 설정

Ubuntu Server 문서는 Apache 설정 파일이 `/etc/apache2/` 아래에 있으며, 사이트별 설정은 `sites-available`과 `sites-enabled` 구조로 관리한다고 설명합니다. [Ubuntu][2]

설정 파일을 생성합니다.

```bash
sudo cp /etc/apache2/sites-available/000-default.conf /etc/apache2/sites-available/opencart.conf
sudo nano /etc/apache2/sites-available/opencart.conf
```

아래처럼 수정합니다.

```apache
<VirtualHost *:80>
    ServerAdmin admin@example.com
    ServerName your-server-ip-or-domain
    DocumentRoot /var/www/opencart

    <Directory /var/www/opencart>
        AllowOverride All
        Require all granted
    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/opencart_error.log
    CustomLog ${APACHE_LOG_DIR}/opencart_access.log combined
</VirtualHost>
```

설정을 적용합니다.

```bash
sudo a2enmod rewrite
sudo a2ensite opencart.conf
sudo a2dissite 000-default.conf
sudo apache2ctl configtest
sudo systemctl reload apache2
```

`AllowOverride All`은 `.htaccess`를 사용하기 위한 설정이며, `a2enmod rewrite`는 OpenCart의 SEO URL 동작에 필요합니다. 또한 Apache 공식 문서는 `apachectl configtest` 또는 `-t` 옵션으로 설정 문법을 검사할 수 있다고 안내합니다. [OpenCart][7] [Apache][8]

## 7단계. 브라우저에서 설치 마무리

이제 브라우저에서 아래 주소로 접속합니다.

```text
http://서버IP/
```

OpenCart 공식 설치 흐름은 다음과 같습니다. [OpenCart][6]

1. 라이선스 동의
2. 사전 요구사항 검사
3. DB 정보와 관리자 계정 입력
4. 설치 완료 후 `install` 폴더 삭제

사전 요구사항 검사 단계에서 `/var/www/opencart/admin/config.php`가 `Missing`으로 표시되면, 5단계의 파일 생성 명령이 누락되었거나 제대로 반영되지 않은 경우가 많습니다. 이때는 아래처럼 파일 존재 여부를 먼저 확인합니다.

```bash
ls -l /var/www/opencart/config.php /var/www/opencart/admin/config.php
```

`admin/config.php`가 없다면 다음 명령으로 다시 생성한 뒤 새로고침합니다.

```bash
sudo cp /var/www/opencart/admin/config-dist.php /var/www/opencart/admin/config.php
sudo chown www-data:www-data /var/www/opencart/admin/config.php
sudo chmod 664 /var/www/opencart/admin/config.php
```

### 설치 화면의 DB 입력값 예시

DB 입력 화면에서는 아래처럼 입력합니다.

* DB Driver: `MySQLi`
* Hostname: `localhost`
* Username: `opencartuser`
* Password: 3단계에서 생성한 **MySQL 사용자 비밀번호**
* Database: `opencart`
* Prefix: `oc_`
* Port: `3306`
* Advanced: 끔

여기서 **Password 칸은 `sudo` 비밀번호가 아니라 MySQL의 `opencartuser` 비밀번호**입니다. 따라서 리눅스 로그인 비밀번호나 root 계정 비밀번호를 넣지 않도록 주의합니다.

현재 생성된 사용자와 DB를 다시 확인하려면 다음 명령을 사용할 수 있습니다.

```bash
sudo mysql -e "select user, host from mysql.user"
sudo mysql -e "show databases"
```

예를 들어 결과에 아래처럼 보이면:

* 사용자: `opencartuser@localhost`
* 데이터베이스: `opencart`

설치 화면에도 그대로 아래처럼 입력하면 됩니다.

* Username: `opencartuser`
* Database: `opencart`

만약 DB 비밀번호를 잊어버렸다면 아래처럼 새 비밀번호로 다시 지정한 뒤, 설치 화면에도 같은 값을 넣습니다.

```bash
sudo mysql
ALTER USER 'opencartuser'@'localhost' IDENTIFIED BY '새비밀번호';
FLUSH PRIVILEGES;
EXIT;
```

보안상 설치 화면의 DB Username에는 `root`보다 **OpenCart 전용 사용자(`opencartuser`)를 사용하는 편이 더 적절**합니다.

### 관리자 계정 입력값 예시

설치 화면 아래쪽의 관리자 계정 정보는 DB 계정과 별개입니다.

* Username: `admin` 또는 원하는 관리자 아이디
* Password: OpenCart 관리자 로그인용 비밀번호
* E-Mail: 본인 이메일 주소

즉, 아래쪽 관리자 계정 비밀번호는 DB 비밀번호와 동일할 필요가 없습니다. 설치가 끝난 뒤 관리자 페이지 로그인에 사용할 값이므로, 따로 기억하기 쉬운 강한 비밀번호를 설정합니다.

관리자 계정도 이 단계에서 생성합니다. OpenCart 공식 문서는 설치 중 **새 `admin` 디렉터리 이름을 지정해 관리자 경로를 바꾸는 것**을 보안 권장사항으로 안내합니다. [OpenCart][6]

## 8단계. 설치 후 보안 마무리

OpenCart 공식 설치 문서는 설치 후 아래를 권장합니다. [OpenCart][6]

* `install` 디렉터리 삭제
* `storage` 폴더를 웹 루트 밖으로 이동
* 설치 중 새 `admin` 디렉터리 이름 사용
* 강한 관리자 비밀번호 사용

가장 먼저 해야 할 작업은 `install` 디렉터리 삭제입니다.

```bash
sudo rm -rf /var/www/opencart/install
```

OpenCart 4 설치 과정에서는 `storage` 폴더를 웹 루트 밖으로 이동하는 옵션을 제공할 수 있으므로, 설치 화면에서 해당 항목이 보이면 적용하는 것이 좋습니다. [OpenCart][6]

# 2) 설치 후 확인할 것

아래 항목을 확인하면 기본 설치가 정상적으로 끝났는지 점검할 수 있습니다.

1. 브라우저에서 `http://서버IP/` 접속 시 OpenCart 메인 페이지가 뜨는지
2. `http://서버IP/admin/` 또는 설치 중 변경한 관리자 경로로 로그인되는지
3. Apache 로그가 정상적으로 기록되는지
4. 관리자 화면에서 상품 또는 카테고리 1개 등록이 되는지

**로그 확인 명령:**

```bash
sudo tail -f /var/log/apache2/opencart_access.log
sudo tail -f /var/log/apache2/opencart_error.log
```

Apache는 사이트 설정과 로그를 `/etc/apache2/` 및 관련 디렉터리 구조로 관리합니다. [Ubuntu][2]
