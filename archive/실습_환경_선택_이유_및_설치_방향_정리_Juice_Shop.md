# Ubuntu 22.04 + OWASP Juice Shop 실습 환경 선택 이유 및 설치 방향 정리

## 1. 문서 개요
이 문서는 Ubuntu 22.04 Server 환경에서 **OWASP Juice Shop** 기반 실습 환경을 구축하기 위해, 운영체제·웹서버·애플리케이션 실행 방식·실습 대상 선택 이유와 설치 방향을 한 번에 정리한 통합 문서이다. 보고서 작성에 바로 활용할 수 있도록 선택 근거, 설치 시 주의사항, 보고서용 문장 예시, 체크포인트까지 함께 정리하였다.

기존에는 OpenCart 기반 쇼핑몰 구축을 실습 대상으로 검토했지만, 현재 실습의 핵심 목적은 **정상 서비스 운영**보다 **웹 공격 탐지, 웹 로그 수집, 공격 유형 분류 실험**에 있다. 따라서 이 문서는 OpenCart 대신 **OWASP Juice Shop**을 기준으로 실습 환경을 다시 정리한 문서이다.

---

## 2. 어떤 Ubuntu ISO를 설치해야 하는가
가상머신(VMware, VirtualBox 등)에 서버를 설치할 목적이라면 **`ubuntu-22.04.5-live-server-amd64.iso`** 를 사용하는 것이 적절하다.

- **Live Server ISO**: 서버 구축 목적에 적합한 이미지
- **Desktop ISO**: GUI 중심의 일반 PC 사용 환경에 적합한 이미지
- **WSL용 이미지**: Windows Subsystem for Linux 환경용으로, 일반적인 가상머신 설치용이 아님

즉, 이번 실습처럼 웹서버와 실습용 웹 애플리케이션을 직접 구축하는 목적이라면 Desktop 이미지보다 **Server 이미지**가 더 적합하다.

## 3. Ubuntu 22.04를 채택한 이유

### 3.1 LTS 버전이라 안정성이 높다
Ubuntu 22.04는 LTS(Long Term Support) 계열이므로 장기 운영과 실습 환경 유지에 적합하다.

- 장기 지원 버전이라 운영 안정성이 높다.
- 표준 보안 유지보수 기간이 길어 실습용 서버로 사용하기에 무리가 적다.
- 서버 운영에서는 일반적으로 LTS 버전 사용이 권장된다.

### 3.2 서버 실습 자료와 공식 문서가 많다
Ubuntu는 Apache, Docker 같은 전형적인 서버 구성 자료가 풍부하다.

- 웹서버 설치와 서비스 관리 방법을 찾기 쉽다.
- 가상호스트 설정, 로그 확인, 패키지 설치 방법 등의 자료가 많다.
- 초보자 입장에서 문제 해결에 참고할 수 있는 문서가 풍부하다.

### 3.3 Juice Shop 실험 환경과 잘 맞는다
OWASP Juice Shop은 Docker로 빠르게 실행하는 방식이 일반적이므로, Ubuntu 22.04 Server 환경과 잘 맞는다.

- Apache를 앞단에 두고 Reverse Proxy 구조를 구성하기 쉽다.
- Docker 기반 애플리케이션 실행 실습과 연결하기 좋다.
- 웹 로그 수집 실험을 위한 최소 구성으로 정리하기 쉽다.

### 3.4 서버 목적에 맞는 구성이 가능하다
이번 실습 목적은 단순 데스크톱 사용이 아니라 **웹서버 구축, 실습용 웹 애플리케이션 설치, 로그 확인, 취약점 점검 실습**에 있다.

- GUI 중심의 Desktop 환경보다 Server 환경이 목적에 더 맞다.
- 불필요한 요소가 적어 서버 실습에 집중하기 좋다.
- 서버 관리 명령, 로그 확인, 서비스 운영 구조를 학습하기 쉽다.

### 3.5 정리
Ubuntu 22.04를 선택한 이유는 단순히 버전 번호 때문이 아니라, 다음과 같은 실습 적합성 때문이다.

- LTS 기반의 안정성
- 서버 구축 자료와 문서의 풍부함
- Apache 및 Docker 기반 구성과의 궁합
- 웹서버 실습 목적에 맞는 서버 중심 구성

## 4. OWASP Juice Shop을 채택한 이유

### 4.1 보안 실습용으로 설계된 무료 오픈소스 애플리케이션이다
OWASP Juice Shop은 무료 오픈소스 웹 애플리케이션이며, 보안 교육과 취약점 실습을 위해 의도적으로 취약하게 설계된 환경이다.

- 라이선스 비용 없이 사용 가능하다.
- 보안 실습과 로그 수집 실험에 적합하다.
- 일반 쇼핑몰 형태의 화면을 가지므로 실습 대상이 직관적이다.

### 4.2 공격 재현과 로그 수집에 유리하다
Juice Shop은 단순한 쇼핑몰 구축용이 아니라, 다양한 웹 취약점 실험을 위해 설계된 애플리케이션이다.

- 여러 종류의 공격성 요청을 한 환경에서 유도하기 쉽다.
- 정상 요청과 공격성 요청을 함께 수집하기 좋다.
- 같은 실험을 반복하기 쉬워 로그 데이터 축적에 유리하다.

즉, 단순 웹사이트 배포 실습보다 **공격 탐지용 데이터 수집 실험**에 더 잘 맞는다.

### 4.3 설치 구조가 비교적 단순하다
Juice Shop은 보통 Docker 컨테이너로 실행하므로 설치 흐름이 비교적 명확하다.

- Docker 설치
- Juice Shop 이미지 다운로드
- 컨테이너 실행
- Apache Reverse Proxy 연결
- 로그 확인

즉, 웹 애플리케이션 배포 흐름과 리버스 프록시 구조를 함께 학습하기 좋다.

### 4.4 실험 목적을 설명하기 쉽다
Juice Shop은 보안 교육과 취약점 실습을 위한 대상으로 잘 알려져 있으므로, 보고서에서 **합법적이고 통제된 실험 환경**이라는 점을 설명하기 쉽다.

- 실습 목적과 윤리적 정당성을 함께 설명하기 좋다.
- 실제 서비스 공격이 아니라 자가 실습 환경임을 분명히 할 수 있다.
- 보고서에서 실험 대상 선택 이유를 설득력 있게 쓸 수 있다.

### 4.5 Apache 환경과 함께 쓰기 좋다
Juice Shop 자체는 Node.js 기반 애플리케이션이지만, 앞단에 Apache를 두고 Reverse Proxy로 연결하면 로그 수집에 특히 유리하다.

- 모든 요청을 Apache access log에 남길 수 있다.
- 필요하면 `combined`보다 풍부한 로그 포맷으로 확장할 수 있다.
- 요청 시간, User-Agent, Referer, 상태코드 등을 분석용 특징값으로 활용하기 쉽다.

### 4.6 정리
OWASP Juice Shop을 선택한 이유는 다음과 같다.

- 무료 오픈소스 실습용 웹 애플리케이션이기 때문
- 공격 재현과 로그 수집에 유리하기 때문
- 설치 구조가 비교적 단순하기 때문
- Apache와 함께 사용해 웹 로그를 수집하기 좋기 때문
- 보안 실습용 대상으로 설명하기 적절하기 때문

## 5. 웹서버 프로그램으로 Apache2를 채택한 이유
Apache2는 Juice Shop 실습 환경에서 **로그 수집과 Reverse Proxy 구성** 측면에서 학습 편의성이 높다.

### 5.1 Juice Shop 앞단 웹서버로 사용하기 좋다
- Apache는 Reverse Proxy 설정이 가능하다.
- 외부 요청을 Apache가 먼저 받고, 내부적으로 Juice Shop으로 전달하는 구조를 만들기 쉽다.
- 웹 애플리케이션 자체와 로그 수집 지점을 분리할 수 있다.

### 5.2 Ubuntu에서 자료가 많고 설치가 쉽다
- Ubuntu 환경에서 Apache 설치와 설정 자료가 풍부하다.
- 기본 패키지 관리 방식으로 손쉽게 설치할 수 있다.
- 초보자 실습 환경에서 접근성이 좋다.

### 5.3 로그 확인과 실습 보고서 작성에 유리하다
- `access.log`, `error.log` 확인이 쉽다.
- 웹 요청 처리 결과를 추적하기 좋다.
- `LogFormat`을 확장해 분석용 로그를 만들기 쉽다.
- 실습 보고서에 “로그 확인 과정”을 쓰기 편하다.

### 5.4 정리
Apache2를 채택한 이유는 다음과 같다.

- Juice Shop 앞단 리버스 프록시로 사용하기 좋음
- Ubuntu에서 설치·설정 자료가 풍부함
- 로그 확인이 쉬워 실습과 보고서 작성에 적합함

---

### 6. 애플리케이션 실행 방식으로 Docker를 채택한 이유

Docker는 Juice Shop을 간단하게 실행하고 재배포하기에 적합한 방식이다.

#### 6.1 설치와 재실행이 간단하다

* Docker 이미지를 내려받아 곧바로 컨테이너로 실행할 수 있다.
* 실습 환경을 다시 만들 때 재현성이 높다.
* 오류가 생겨도 컨테이너를 삭제하고 다시 띄우기 쉽다.

#### 6.2 실습용 환경 분리에 유리하다

* 애플리케이션을 운영체제에 직접 복잡하게 설치하지 않아도 된다.
* 서비스 중지, 재시작, 삭제가 비교적 단순하다.
* 실습 환경을 정리하거나 초기화하기 쉽다.

#### 6.3 Apache와 역할을 분리하기 좋다

* Apache는 앞단 웹서버와 로그 수집 역할을 담당한다.
* Juice Shop은 Docker 컨테이너 안에서 애플리케이션 역할을 담당한다.
* 이 구조는 서비스 계층과 로그 수집 계층을 구분하는 데 도움이 된다.

#### 6.4 정리

Docker를 채택한 이유는 다음과 같다.

* Juice Shop 실행이 간단하기 때문
* 재현성과 복구 편의성이 높기 때문
* Apache와 애플리케이션의 역할 분리가 쉽기 때문

---

### 7. 애플리케이션 실행 환경

Juice Shop 실습 환경에서는 PHP와 MySQL을 직접 구성하기보다, **Docker 기반 실행 환경**을 사용하는 편이 더 적합하다.

#### 7.1 핵심 구성 요소

* `docker.io`: Docker 엔진 패키지
* Apache Reverse Proxy 모듈: `proxy`, `proxy_http`
* 추가 로그 확장 모듈: `headers`, `logio`, `unique_id`

#### 7.2 각 구성 요소의 역할

* `docker.io`: Juice Shop 컨테이너 실행
* `proxy`, `proxy_http`: Apache가 내부의 Juice Shop으로 요청 전달
* `headers`: 프록시 환경에서 헤더 처리 보조
* `logio`: 실제 입출력 바이트 로그 수집
* `unique_id`: 요청별 식별자 생성

#### 7.3 정리

이 구성은 Apache 웹서버가 외부 요청을 받아 로그를 기록하고, 내부적으로 Docker 컨테이너의 Juice Shop으로 전달하는 기본 환경이다.

---

### 8. 최종 실습 환경 구성

이번 실습 환경은 아래와 같이 정리할 수 있다.

* **운영체제**: Ubuntu 22.04 LTS Server
* **웹서버**: Apache2
* **프록시/로그 모듈**: `proxy`, `proxy_http`, `headers`, `logio`, `unique_id`
* **애플리케이션 실행 방식**: Docker
* **실습 대상 애플리케이션**: OWASP Juice Shop

이 조합을 선택한 이유는 다음과 같다.

1. Ubuntu 22.04 LTS는 안정적인 서버 실습 환경을 제공한다.
2. Apache2와 Docker 구성이 Ubuntu에서 비교적 쉽다.
3. Juice Shop은 이 조합과 잘 맞고 빠르게 실행할 수 있다.
4. 무료 기반의 실습 환경으로 구축 부담이 크지 않다.
5. 웹서버 로그 확인, 공격 요청 수집, 보안 점검 실습까지 연결하기 좋다.

---

### 9. 실습 흐름 요약

실습 흐름은 다음과 같이 정리할 수 있다.

1. Ubuntu 22.04 Server를 가상머신에 설치한다.
2. Apache2를 설치한다.
3. Docker를 설치한다.
4. Juice Shop 이미지를 내려받고 컨테이너를 실행한다.
5. Apache Reverse Proxy 설정을 적용한다.
6. Apache access log와 error log를 설정한다.
7. 브라우저에서 Juice Shop 화면이 열리는지 확인한다.
8. 허가된 실험 도구 또는 수동 테스트로 요청을 발생시킨다.
9. 웹서버 로그를 확인하면서 동작을 검증한다.

한 줄로 요약하면 다음과 같다.

> Ubuntu 22.04 Server를 가상머신에 설치하고, Apache2 + Docker 환경을 구성한 뒤, OWASP Juice Shop을 Reverse Proxy 구조로 연결하여 웹 로그를 수집하고 동작을 검증한다.

---

### 10. 설치 시 핵심 체크포인트

설치 과정에서 특히 확인해야 할 핵심 항목은 다음과 같다.

* 설치 ISO는 `ubuntu-22.04.5-live-server-amd64.iso` 사용
* Apache 설치 후 서비스 상태 확인
* Docker 설치 후 `docker ps` 동작 확인
* Juice Shop 컨테이너는 `127.0.0.1:3000`에 바인딩하여 외부 직접 노출을 피함
* Apache에서 `ProxyPass`, `ProxyPassReverse` 설정 적용
* `a2enmod proxy proxy_http headers logio unique_id` 적용
* `apache2ctl configtest`로 설정 문법 확인
* `/var/log/apache2/` 아래 access log, error log 생성 확인
* 브라우저 접속 시 Apache 기본 페이지가 아니라 Juice Shop 화면이 보여야 함
* 점검 도구 실행 시 access log가 증가하는지 확인

### 11. 보고서용 문단 예시

> 본 실습에서는 Ubuntu 22.04 LTS Server를 기반으로 Apache2, Docker, OWASP Juice Shop을 조합하여 웹 로그 수집 및 공격 탐지 실험 환경을 구성하였다. Ubuntu 22.04 LTS는 장기 지원 기반의 안정적인 서버 운영체제이며, Apache2는 Ubuntu 환경에서 설치와 설정 자료가 풍부하고 로그 확인이 쉽다. OWASP Juice Shop은 보안 실습과 공격 재현을 위해 설계된 웹 애플리케이션이므로, 다양한 공격성 요청을 발생시켜 웹 로그를 수집하는 목적에 적합하다. 실습에서는 Apache를 앞단 웹서버로 사용하고, Juice Shop은 Docker 컨테이너로 실행한 뒤 Reverse Proxy로 연결하여 모든 요청이 Apache 로그에 기록되도록 구성하였다.

### 12. 보고서용 짧은 문장 예시

* Ubuntu 22.04 LTS를 선택한 이유는 장기 지원 기반의 안정성과 서버 실습 자료의 풍부함 때문이다.
* OWASP Juice Shop을 선택한 이유는 공격 재현과 로그 수집에 적합한 실습용 웹 애플리케이션이기 때문이다.
* Apache2를 선택한 이유는 Reverse Proxy 구성과 로그 확인이 쉽기 때문이다.
* Docker를 선택한 이유는 Juice Shop 실행과 재배포가 간단하고 재현성이 높기 때문이다.
* 이번 실습 환경은 Apache2가 외부 요청을 받고, 내부의 Juice Shop으로 전달하는 구조로 구성하였다.
