# 01_운영_기준_실행_가이드

- 문서 상태: 운영 기준 문서
- 목적: 현재 실제 운영/실행 기준의 single source of truth

이 문서가 실제 운영 명령, 경로, OpenAI/Claude 실행 차이, 수동 절차의 기준 문서다.  
다른 문서와 표현이 다르면 실제 운영은 이 문서를 우선한다.

## 1. 목적

- 실제 운영자가 그대로 따라 할 수 있는 절차를 한 문서에 모은다.
- 흩어진 명령과 경로 기준을 현재 운영 기준으로 통일한다.
- OpenAI와 Claude 실행 차이를 한곳에서 확인하게 한다.

## 2. 현재 실제 디렉터리 구조

기준 작업 루트는 `/opt/web_log_analysis`다.

```text
/opt/web_log_analysis/
├── config/
│   └── llm.env
├── data/
│   ├── raw/
│   └── processed/
├── reports/
├── logs/
├── src/
└── .venv/
```

- raw 입력: `/opt/web_log_analysis/data/raw`
- 전처리 결과: `/opt/web_log_analysis/data/processed`
- 보고서 결과: `/opt/web_log_analysis/reports`
- 실행 로그: `/opt/web_log_analysis/logs`

주의:

- `export_db_logs_cli.py`는 기본적으로 `data/raw/`에 출력하므로 현재 운영 기준과 맞는다.
- `prepare_llm_input.py`, `llm_stage1_classifier.py`, `llm_stage2_reporter.py`는 운영 기준 경로를 맞추기 위해 `--out-dir`를 명시한다.
- `run_analysis_pipeline.py` 기본값은 `<work-dir>/data/processed`와 `<work-dir>/reports`다. 현재 운영 기준과 맞으므로 표준 pipeline 실행에서는 별도 디렉터리 옵션을 생략한다.

## 3. 서버 역할 구분

- 웹서버: Apache 로그 생성, shipper 실행
- DB 서버: `web_logs` 저장
- LLM 서버: export, prepare, stage1, stage2, pipeline 실행

운영 기준:

- export는 DB 서버가 아니라 LLM 서버에서 수행한다.
- routine 분석의 기본 입력은 `security` export다.

## 4. env 파일 사용법

기준 파일은 `/opt/web_log_analysis/config/llm.env`다.

예시:

```dotenv
OPENAI_API_KEY=실제_OpenAI_API_KEY
OPENAI_BASE_URL=https://api.openai.com/v1

# 기본 provider는 openai다.
LLM_PROVIDER=openai

# Claude를 사용할 때만 활성화
# ANTHROPIC_API_KEY=실제_Anthropic_API_KEY
# ANTHROPIC_BASE_URL=https://api.anthropic.com/v1
# ANTHROPIC_MODEL=claude_모델명

LOG_DB_HOST=DB서버_IP
LOG_DB_PORT=3306
LOG_DB_NAME=web_logs
LOG_DB_USER=log_writer
LOG_DB_PASSWORD=실제_DB_조회_비밀번호
KNOWN_ASSET_IPS=필요시_자산_IP_목록
```

기준:

- provider 미지정 시 기본은 `openai`
- Claude 사용 시 `LLM_PROVIDER=anthropic`, `ANTHROPIC_API_KEY`, `ANTHROPIC_MODEL` 필요
- 사용하지 않는 provider 키는 주석 처리 가능
- `LOG_DB_HOST`와 `LOG_DB_PASSWORD`는 운영 환경에 맞게 환경변수로 지정한다.
- `LOG_DB_USER` 기본값은 `log_writer` 기준이다.
- `KNOWN_ASSET_IPS`와 `--known-asset-ips`는 현재 운영 기준의 필수 항목이 아님

적용:

```bash
cd /opt/web_log_analysis
source .venv/bin/activate
set -a
source ./config/llm.env
set +a
```

## 5. OpenAI / Claude 선택 기준

- OpenAI: provider 옵션을 생략하거나 `LLM_PROVIDER=openai`
- Claude stage1/stage2: `--provider anthropic --model "$ANTHROPIC_MODEL"`
- Claude pipeline: `--llm-provider anthropic --stage1-model "$ANTHROPIC_MODEL" --stage2-model "$ANTHROPIC_MODEL"`

## 6. 공격 또는 테스트 후 수동 절차

### 6.1 shipper

웹서버에서 실행:

```bash
sudo python3 /opt/apache_log_shipper.py
```

확인 포인트:

- `/var/log/apache2/apache_log_shipper.log` 반복 오류 여부
- DB row 증가 여부
- spool 누적 여부

### 6.2 export

LLM 서버에서 실행:

```bash
cd /opt/web_log_analysis
source .venv/bin/activate
set -a
source ./config/llm.env
set +a

python ./src/export_db_logs_cli.py \
  --host "$LOG_DB_HOST" \
  --port "$LOG_DB_PORT" \
  --user "$LOG_DB_USER" \
  --password "$LOG_DB_PASSWORD" \
  --date 2026-04-23 \
  --table security \
  --pretty
```

생성 위치:

- `/opt/web_log_analysis/data/raw/security_2026-04-23_kst.json`

### 6.3 prepare

```bash
python ./src/prepare_llm_input.py \
  --input ./data/raw/security_2026-04-23_kst.json \
  --out-dir ./data/processed \
  --pretty \
  --write-filtered-out
```

생성 위치:

- `/opt/web_log_analysis/data/processed/security_2026-04-23_kst_llm_input.json`
- `/opt/web_log_analysis/data/processed/security_2026-04-23_kst_analysis_candidates.json`
- `/opt/web_log_analysis/data/processed/security_2026-04-23_kst_noise_summary.json`
- `/opt/web_log_analysis/data/processed/security_2026-04-23_kst_filtered_out_rows.json`

### 6.4 stage1

OpenAI:

```bash
python ./src/llm_stage1_classifier.py \
  --input ./data/processed/security_2026-04-23_kst_llm_input.json \
  --out-dir ./data/processed \
  --mode routine \
  --pretty
```

Claude:

```bash
python ./src/llm_stage1_classifier.py \
  --provider anthropic \
  --model "$ANTHROPIC_MODEL" \
  --input ./data/processed/security_2026-04-23_kst_llm_input.json \
  --out-dir ./data/processed \
  --mode routine \
  --pretty
```

### 6.5 stage2

OpenAI:

```bash
python ./src/llm_stage2_reporter.py \
  --stage1-results ./data/processed/security_2026-04-23_kst_stage1_results.json \
  --llm-input ./data/processed/security_2026-04-23_kst_llm_input.json \
  --out-dir ./reports \
  --mode routine \
  --pretty
```

Claude:

```bash
python ./src/llm_stage2_reporter.py \
  --provider anthropic \
  --model "$ANTHROPIC_MODEL" \
  --stage1-results ./data/processed/security_2026-04-23_kst_stage1_results.json \
  --llm-input ./data/processed/security_2026-04-23_kst_llm_input.json \
  --out-dir ./reports \
  --mode routine \
  --pretty
```

기준:

- 현재 운영 기준 명령 세트에서는 `--known-asset-ips`를 기본 명령에서 넣지 않는다.
- 필요하면 선택적으로만 추가한다.

### 6.6 pipeline

OpenAI:

```bash
python ./src/run_analysis_pipeline.py \
  --export-input ./data/raw/security_2026-04-23_kst.json \
  --work-dir /opt/web_log_analysis \
  --mode routine \
  --pretty
```

Claude:

```bash
python ./src/run_analysis_pipeline.py \
  --llm-provider anthropic \
  --stage1-model "$ANTHROPIC_MODEL" \
  --stage2-model "$ANTHROPIC_MODEL" \
  --export-input ./data/raw/security_2026-04-23_kst.json \
  --work-dir /opt/web_log_analysis \
  --mode routine \
  --pretty
```

기본 경로와 다르게 저장해야 할 때만 `--processed-dir` 또는 `--reports-dir`를 선택적으로 지정한다.

## 7. 산출물 위치

## 10. 자주 헷갈리는 부분

- export는 LLM 서버에서 실행한다.
- prepare와 stage1 결과는 `./data/processed` 기준이다. `./processed`가 아니다.
- stage2 결과는 `./reports` 기준이다.
- pipeline 기본값은 `./data/processed`와 `./reports` 기준이므로 운영 경로와 맞는다.
- provider를 지정하지 않으면 기본은 OpenAI다.
- `KNOWN_ASSET_IPS`는 현재 운영 기준의 필수 항목이 아니다.

## 11. 짧은 순서 요약

요청 발생  
-> Apache 로그 생성  
-> shipper DB 적재  
-> LLM 서버 export  
-> prepare  
-> stage1  
-> stage2 또는 pipeline  
-> 보고서 회수
