# 00_전체_흐름_요약_가이드

- 문서 상태: 정리본
- 버전: v0.4
- 작성일: 2026-04-09

## 1. 한눈에 보는 구조

```text
웹서버
  - Apache
  - app_access.log / app_security.log / app_error.log
  - apache_log_shipper.py
        ↓
DB 서버
  - MariaDB
  - web_logs
        ↓
LLM 서버
  - export_db_logs_cli.py
  - prepare_llm_input.py
  - llm_stage1_classifier.py
  - llm_stage2_reporter.py
  - run_analysis_pipeline.py
```

## 2. 서버별 역할

- 웹서버: 로그 생성, shipper 실행
- DB 서버: 로그 저장
- LLM 서버: export, 전처리, stage1, stage2 실행

## 3. 로그 역할

- `security`: 분석 기본 입력
- `error`: 예외/5xx 상관분석 보조
- `access`: 운영 확인용

## 4. 현재 실행 순서

1. 웹서버에서 Apache 로그 생성
2. shipper가 DB 적재
3. LLM 서버에서 `security` 중심 export
4. `prepare_llm_input.py` 실행
5. `llm_stage1_classifier.py` 실행
6. `llm_stage2_reporter.py` 실행

## 5. 시간 기준

- DB 저장: UTC
- 사용자 입력: KST
- export 출력: KST

## 6. 파일명 기준

- export: `data/raw/{table}_{date}_kst.json` 또는 `data/raw/{table}_{start}_to_{end}_kst.json`
- prepare: `<base>_llm_input.json`
- stage1: `<base>_stage1_results.json`
- stage2: `<base>_stage2_report.md`

## 7. 현재 보수 해석 기준

path traversal 계열에서는 아래 필드를 우선 본다.

- `resp_content_type`
- `response_body_bytes`
- `raw_request_target`
- `path_normalized_from_raw_request`
- `likely_html_fallback_response`

`resp_html_*` 계열은 현재 보류 기능으로 본다.
