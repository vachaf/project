# 00_전체_흐름_요약_가이드

- 문서 상태: 요약 문서
- 목적: 아키텍처, 서버 역할, 큰 흐름만 빠르게 확인한다.

실제 운영 명령과 절차는 [docs/01_운영_기준_실행_가이드.md](docs/01_운영_기준_실행_가이드.md)를 기준으로 본다.

## 1. 한눈에 보는 구조

```text
웹서버
  - Apache
  - app_access.log / app_security.log / app_error.log
  - apache_log_shipper.py
        ↓
DB 서버
  - MariaDB
  - web_logs
        ↓
LLM 서버
  - export_db_logs_cli.py
  - prepare_llm_input.py
  - llm_stage1_classifier.py
  - llm_stage2_reporter.py
  - run_analysis_pipeline.py
```

## 2. 서버별 역할

- 웹서버: Apache 로그 생성, shipper 실행, spool 상태 확인
- DB 서버: `web_logs` 보관, 조회 계정 제공
- LLM 서버: export, prepare, stage1, stage2, 보고서 회수

## 3. 로그 역할

- `security`: 분석 기본 입력
- `error`: 예외와 5xx 상관분석 보조
- `access`: 운영 확인과 기준선 비교용

## 4. 큰 흐름

1. 웹서버에서 로그 생성
2. shipper가 DB 적재
3. LLM 서버에서 `security` 기준 export
4. prepare 실행
5. stage1 실행
6. stage2 실행 또는 pipeline 실행
7. 보고서 회수

## 5. 시간 기준

- DB 저장 시각: UTC
- 사용자 입력과 export 출력: KST

## 6. 현재 운영 기준 경로

- raw 입력: `/opt/web_log_analysis/data/raw`
- 전처리 결과: `/opt/web_log_analysis/data/processed`
- 보고서 결과: `/opt/web_log_analysis/reports`
- 실행 로그: `/opt/web_log_analysis/logs`
