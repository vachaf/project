# A세트 비교 실험 실행 요청 세트 (Juice Shop 기준)

작성 기준일: 2026-04-24  
목적: Juice Shop 정상 baseline 및 인증/로그인 패턴 비교 실험용 raw log를 생성한다.  
적용 범위: `docs/98_비교_실험_요청_세트_표준.md`를 따르는 A세트 실행 문서

이 문서는 상위 표준안 기준의 A세트 실행 문서다. A세트는 정상 baseline + 인증/로그인 패턴 비교에 집중한다. SQLi / XSS / Traversal / HPP / Dir Probe 는 별도 세트에서 수행한다.

---

# 1. 실행 환경

BASE_URL=http://127.0.0.1:3000  
UA_PREFIX=lab-a-set

모든 요청은 curl 기반 CLI 실행을 기준으로 한다. 비교 방식은 상위 표준안의 full pipeline / stage1 / stage2 정의를 따른다.

---

# 2. 실행 목적

A세트는 다음 목적을 가진다:

- 정상 트래픽 baseline 생성
- 검색 / 조회 / 인증 패턴 확보
- benign vs suspicious_auth_abuse 경계 확인
- stage1 / stage2 provider 비교용 동일 input 생성

provider 비교 시에는 같은 raw log, 같은 `known_asset_ips`, 같은 top-N 조건을 유지한다. 가능하면 stage2 비교는 같은 `*_stage1_results.json` 을 고정한 비교도 별도로 수행한다. `known_asset_ips` 는 raw log 를 생성한 환경 기준으로 맞춘다.

---

# 3. 정상 기준선 세트

## A-01 Home
```bash
curl -i -A "${UA_PREFIX}-home-1" "$BASE_URL/"
```
- 기대 ground truth: 정상 baseline
- 기대 관찰 포인트: 후보 밖 정상 처리 또는 일반 정상 트래픽 분류 여부

## A-02 Product List
```bash
curl -i -A "${UA_PREFIX}-products-1" "$BASE_URL/rest/products"
```
- 기대 ground truth: 정상 baseline
- 기대 관찰 포인트: 조회형 요청을 공격 후보로 과도하게 올리지 않는지

## A-03 Search apple
```bash
curl -i -A "${UA_PREFIX}-search-apple-1" "$BASE_URL/rest/products/search?q=apple"
```
- 기대 ground truth: 정상 baseline
- 기대 관찰 포인트: `benign_normal_search` 또는 후보 밖 정상 처리 여부

## A-04 Search phone
```bash
curl -i -A "${UA_PREFIX}-search-phone-1" "$BASE_URL/rest/products/search?q=phone"
```
- 기대 ground truth: 정상 baseline
- 기대 관찰 포인트: 일반 검색을 공격성 검색과 어떻게 구분하는지

## A-05 Search bag
```bash
curl -i -A "${UA_PREFIX}-search-bag-1" "$BASE_URL/rest/products/search?q=bag"
```
- 기대 ground truth: 정상 baseline
- 기대 관찰 포인트: provider별 오탐 여부와 normal_search 해석 차이

## A-06 Product Detail
```bash
curl -i -A "${UA_PREFIX}-product-1-1" "$BASE_URL/rest/products/1"
```
- 기대 ground truth: 정상 baseline
- 기대 관찰 포인트: 단순 상품 상세 조회를 정상 조회로 유지하는지

## A-07 WhoAmI
```bash
curl -i -A "${UA_PREFIX}-whoami-1" "$BASE_URL/rest/user/whoami"
```
- 기대 ground truth: 정상 baseline
- 기대 관찰 포인트: 비인증 상태의 정상 API 확인 요청을 오탐하지 않는지

## A-08 Mixed browsing patterns
```bash
curl -i -A "${UA_PREFIX}-browse-apple-1" "$BASE_URL/rest/products/search?q=apple"
curl -i -A "${UA_PREFIX}-browse-phone-1" "$BASE_URL/rest/products/search?q=phone"
curl -i -A "${UA_PREFIX}-browse-bag-1" "$BASE_URL/rest/products/search?q=bag"
```
- 기대 ground truth: 정상 baseline
- 기대 관찰 포인트: 짧은 연속 탐색을 benign browsing 으로 유지하는지

# 4. 인증 / 로그인 세트

## A-09 Login Fail (single)
```bash
curl -i \
  -A "${UA_PREFIX}-login-fail-1" \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@juice-sh.op","password":"1234"}' \
  "$BASE_URL/rest/user/login"
```
- 기대 ground truth: 단일 로그인 실패
- 기대 관찰 포인트: `suspicious_auth_abuse` 와 `likely_false_positive` 사이에서 어떻게 해석하는지

## A-10 Login Fail (repeat x3)
```bash
curl -i -A "${UA_PREFIX}-login-repeat-1" -H "Content-Type: application/json" -d '{"email":"admin@juice-sh.op","password":"1234"}' "$BASE_URL/rest/user/login"
sleep 2
curl -i -A "${UA_PREFIX}-login-repeat-2" -H "Content-Type: application/json" -d '{"email":"admin@juice-sh.op","password":"1234"}' "$BASE_URL/rest/user/login"
sleep 2
curl -i -A "${UA_PREFIX}-login-repeat-3" -H "Content-Type: application/json" -d '{"email":"admin@juice-sh.op","password":"1234"}' "$BASE_URL/rest/user/login"
```
- 기대 ground truth: 반복 로그인 실패
- 기대 관찰 포인트: 반복성 때문에 provider가 단일 실패보다 더 공격적으로 해석하는지

## A-11 Login Bypass Success (Juice Shop)
```bash
curl -i \
  -A "${UA_PREFIX}-login-bypass-1" \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@juice-sh.op'\'' -- ","password":"x"}' \
  "$BASE_URL/rest/user/login"
```
- 기대 ground truth: 성공한 인증 우회
- 기대 관찰 포인트: stage1/stage2가 단순 로그인 실패보다 더 강하게 해석하는지, 보고서가 200 응답 / token 발급 / admin role 등 성공 흔적을 어떻게 서술하는지

## A-12 Login Success (optional)
```bash
curl -i \
  -A "${UA_PREFIX}-login-success-1" \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@juice-sh.op","password":"admin123"}' \
  "$BASE_URL/rest/user/login"
```
- 기대 ground truth: 정상 인증 성공
- 기대 관찰 포인트: 정상 로그인 성공을 오탐하지 않는지

## A-13 Cart / Basket access (optional)
```bash
curl -i -A "${UA_PREFIX}-cart-1" "$BASE_URL/rest/basket/1"
```
- 기대 ground truth: 인증 상태 확인 또는 비인증 보호 API 접근 패턴 확인
- 기대 관찰 포인트: 인증 상태 API / 보호 리소스 접근을 어떤 수준의 신호로 해석하는지

---

# 5. 실행 목적 요약

이 A세트는 baseline/auth 전용 실행 세트다. 정상 검색/조회 baseline 과 인증 실패, 반복 실패, bypass 성공, 정상 성공(optional) 을 분리해 provider별 stage1 / stage2 차이를 비교한다.

---

# 6. 사용 방식

1. Juice Shop 환경에서 요청을 실행해 raw log를 생성한다.
2. 같은 raw log 기준으로 export, prepare, stage1, stage2를 수행한다.
3. full pipeline 비교 후 필요하면 같은 `*_llm_input.json` 또는 같은 `*_stage1_results.json` 고정 비교를 추가한다.

---

# 7. 주의사항

- A세트는 baseline/auth 전용이며 SQLi / XSS / Traversal / HPP / Dir Probe 는 별도 세트에서 수행한다.
- User-Agent는 반드시 요청별로 구분하고, 요청 간격은 상위 표준안의 운영 주의사항을 따른다.
- stage2 보고서는 사건 요약이므로 suspicious incident는 request_id 기준 raw evidence와 함께 확인한다.
