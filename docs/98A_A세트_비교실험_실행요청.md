# A세트 비교 실험 실행 요청 세트 (Juice Shop 기준)

작성 기준일: 2026-04-24  
목적: Juice Shop 정상 트래픽 baseline 생성 및 stage1 / stage2 provider 비교 실험용 데이터 생성  
적용 범위: OpenAI / Anthropic 동일 raw log 비교 실험

---

# 1. 실행 환경

BASE_URL=http://127.0.0.1:3000  
UA_PREFIX=lab-a-set

모든 요청은 curl 기반 CLI 실행을 기준으로 한다.

---

# 2. 실행 목적

A세트는 다음 목적을 가진다:

- 정상 트래픽 baseline 생성
- 검색 / 조회 / 인증 실패 패턴 확보
- benign vs suspicious 분류 기준 확보
- stage1 / stage2 provider 비교용 동일 input 생성

---

# 3. 정상 기준선 세트 (Baseline Traffic)

## A-01 Home
curl -i -A "${UA_PREFIX}-home-1" "$BASE_URL/"

## A-02 Product List
curl -i -A "${UA_PREFIX}-products-1" "$BASE_URL/rest/products"

## A-03 Search apple
curl -i -A "${UA_PREFIX}-search-apple-1" "$BASE_URL/rest/products/search?q=apple"

## A-04 Search phone
curl -i -A "${UA_PREFIX}-search-phone-1" "$BASE_URL/rest/products/search?q=phone"

## A-05 Search bag
curl -i -A "${UA_PREFIX}-search-bag-1" "$BASE_URL/rest/products/search?q=bag"

## A-06 Product Detail
curl -i -A "${UA_PREFIX}-product-1-1" "$BASE_URL/rest/products/1"

## A-07 WhoAmI
curl -i -A "${UA_PREFIX}-whoami-1" "$BASE_URL/rest/user/whoami"

---

# 4. 인증 / 로그인 패턴 세트

## A-08 Login Fail (single)
curl -i \
  -A "${UA_PREFIX}-login-fail-1" \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@juice-sh.op","password":"1234"}' \
  "$BASE_URL/rest/user/login"

---

## A-09 Login Fail (repeat x3)
curl -i -A "${UA_PREFIX}-login-repeat-1" -H "Content-Type: application/json" -d '{"email":"admin@juice-sh.op","password":"1234"}' "$BASE_URL/rest/user/login"
sleep 2
curl -i -A "${UA_PREFIX}-login-repeat-2" -H "Content-Type: application/json" -d '{"email":"admin@juice-sh.op","password":"1234"}' "$BASE_URL/rest/user/login"
sleep 2
curl -i -A "${UA_PREFIX}-login-repeat-3" -H "Content-Type: application/json" -d '{"email":"admin@juice-sh.op","password":"1234"}' "$BASE_URL/rest/user/login"

---

## A-10 Login Success (optional)
curl -i \
  -A "${UA_PREFIX}-login-success-1" \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@juice-sh.op","password":"admin123"}' \
  "$BASE_URL/rest/user/login"

# 5. 인증 상태 API

## A-11 Cart / Basket access
curl -i -A "${UA_PREFIX}-cart-1" "$BASE_URL/rest/basket/1"

---

# 7. 실행 목적 정리

이 A세트는 다음을 위한 기준 데이터 생성용이다:

- normal_search 분류 기준 확보
- auth failure 반복 패턴 분석
- provider별 false positive 비교
- stage1 severity 차이 분석
- stage2 incident narrative 차이 분석

---

# 8. 사용 방식

1. Juice Shop VM에서 실행하여 로그 생성
2. DB 또는 shipper로 로그 적재
3. LLM 서버에서 export 실행
4. prepare → stage1 → stage2 수행
5. provider별 결과 비교

---

# 9. 주의사항

- SQLi / XSS / Traversal은 포함하지 않는다 (B/C 세트로 분리)
- 동일 환경에서 반복 실행하여 비교 기준을 유지한다
- User-Agent는 반드시 실험별로 구분한다
- stage2는 요약 보고서이며 raw evidence는 stage1 기준으로 확인한다
