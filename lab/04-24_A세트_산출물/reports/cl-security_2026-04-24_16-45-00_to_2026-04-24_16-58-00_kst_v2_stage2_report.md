## 8. 권고 조치
- **P1** 192.168.56.110의 /rest/user/login POST 요청에 대한 원본 로그를 즉시 검토하고, HTTP 200 응답 2건(16:48:05, 16:55:08)에 대해 실제 발급된 JWT/세션 토큰이 무엇인지, 어떤 계정으로 로그인되었는지 확인하라.
  - 근거: 799바이트 application/json 응답은 로그인 성공 후 토큰이 포함된 응답 형태에 부합한다. 인가되지 않은 계정으로 토큰이 발급되었다면 즉각적인 세션 무효화가 필요하다.
- **P1** 192.168.56.110의 테스트 활동이 공식 보안 테스트 계획에 포함된 것인지 담당자(또는 테스트 팀)에게 즉시 확인하라.
  - 근거: User-Agent에 'lab-*' 명칭이 명시되어 있어 내부 실험 트래픽일 가능성이 있으나, 허가되지 않은 자동화 도구 실행일 수도 있다. 확인 전까지는 인증 우회 성공 가능성을 배제할 수 없다.
- **P2** 16:47~16:55 구간의 /rest/user/login 요청 전체를 시계열 순으로 재구성하여, 실패(401) 시퀀스 이후 성공(200)이 이어지는 패턴인지 상관분석하라.
  - 근거: 실패-성공 시퀀스가 짧은 시간(수 초~수 분) 내에 연속 발생하는 패턴은 자격증명 조합 반복 후 우회 성공 흐름일 수 있으며, 단순 오류 테스트와 구분이 필요하다.
- **P2** 후보 밖 low_signal_fuzzing 12건의 출발지 IP·URI·시각을 원본 로그에서 확인하고, 192.168.56.110과의 연계 여부를 검토하라.
  - 근거: 동일 IP·동일 시간대의 저신호 퍼징이 고신호 incident와 결합되면 사전 정찰 단계로 해석될 수 있어 incident 승격 검토 대상이 된다.
- **P2** 인증 엔드포인트(/rest/user/login)에 대한 비브라우저 클라이언트(Referer 없음, 자동화 UA) 접근에 대해 속도 제한(rate limiting) 또는 IP 기반 접근 제어 정책이 적용되어 있는지 점검하라.
  - 근거: 현재 자동화 클라이언트의 반복 인증 시도가 제한 없이 허용된 정황이 있으며, 비인가 자동화 도구에 의한 인증 우회 시도를 조기에 차단할 수 있어야 한다.
- **P3** known_asset_ips(192.168.56.105/108/109/110)에서 발생하는 보안 민감 엔드포인트 접근에 대해 테스트 트래픽 식별 규칙(예: 특정 헤더, 별도 로그 태깅)을 도입하여 실제 위협과 허가된 테스트를 자동으로 구분할 수 있도록 하라.
  - 근거: 현재는 User-Agent 명칭에 의존해 수동으로 구분하는 구조여서, 명칭이 유사한 비인가 도구가 사용될 경우 오탐·미탐 위험이 있다.
- **P3** Socket.IO 폴링 트래픽(192.168.56.1, 62건)을 로그 전처리 단계에서 자동 분리하여 분석 효율을 높이고, 향후 비정상 폴링 패턴(비정상 UA, 비정상 빈도)을 별도 탐지 규칙으로 관리하라.

## 9. 신뢰도와 한계
- 본 보고서는 전처리 및 1차 분류가 완료된 요약 데이터를 기반으로 작성되었으며, 원본 DB 로그 및 실제 요청 페이로드(자격증명 내용, 발급된 토큰 값 등)는 포함되지 않았다. HTTP 200 응답이 실제 인증 성공을 의미하는지 여부는 원본 로그 및 애플리케이션 응답 본문 확인이 필요하다.
- 192.168.56.110은 known_asset_ips에 등록된 내부 대역 IP로, 7건 전체가 이 IP에서 발생하였다. 모든 incident의 confidence가 'medium'이므로 공격자 확정 판단 전 내부 테스트 계획과의 대조 확인이 선행되어야 한다. 허가된 테스트로 확인될 경우 전체 incident가 오탐으로 처리될 수 있다.
- User-Agent 기반 분류는 자동화 도구 식별의 보조 지표이며, UA 문자열은 위변조가 가능하다. 'lab-*' 명칭이 실제 내부 테스트 규약과 일치하는지 외부에서 검증하기 어렵다.
- low_signal_fuzzing 12건의 출발지 IP 및 상세 URI는 본 요약 데이터에 포함되지 않아, 192.168.56.110과의 연계 여부를 현 단계에서 확정할 수 없다. 원본 로그 대조가 필요하다.
- 분석 구간이 13분으로 짧아 장기 패턴(일별·주별 반복 여부, 동일 계정 대상 집중 시도 등)을 판단하기 어렵다. 확장 구간 상관분석이 권고된다.

## 10. 발표용 한 줄 정리
이번 13분 구간에서 주목할 사건은 단 하나의 내부 IP(192.168.56.110)가 자동화 도구를 이용해 인증 엔드포인트를 반복 시도하고, 그 중 2건에서 HTTP 200(로그인 성공 정황)을 받은 것이다. 해당 IP가 known_asset 목록에 포함된 실험 대역임을 감안할 때 허가된 보안 테스트일 가능성이 있으나, 실제 인가 여부가 확인되기 전까지는 인증 우회 성공 가능성을 배제할 수 없다. 외부 공격자 흔적은 이번 구간에서 관측되지 않았으며, 최우선 조치는 테스트 계획 확인과 발급된 세션 토큰 검증이다.
