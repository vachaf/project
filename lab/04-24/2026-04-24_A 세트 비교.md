## 0. 실험 개요

이번 실험은 `docs/98A` 기준 **A세트(baseline/auth 전용)** 비교 실험이다.  
범위는 정상 baseline, 로그인 실패, 반복 로그인 실패, Juice Shop authentication bypass 성공 요청까지이며, SQLi / XSS / Traversal / HPP / Dir Probe 계열은 포함하지 않는다.

실험 방식은 다음과 같다.

- 동일 raw export 사용
- 동일 time window 사용
- 동일 `known_asset_ips` 사용
- 동일 `mode=routine` 사용
- OpenAI / Anthropic 각각에 대해 prepare → stage1 → stage2 비교

이번 라운드는 `v2` 산출물 기준이며, 이전 라운드와 달리 **성공한 HTTP 200 JSON 로그인 응답 2건이 candidate로 승격**되도록 전처리 규칙이 보강되었다. 이로 인해 이전 5건-only 결과와 달리, 이번 비교에서는 **401 실패 5건 + 200 성공 2건**을 함께 비교할 수 있게 되었다.

## 2. 기대값

### 기대 ground truth
- baseline 요청은 과도하게 incident로 승격하지 않는다.
- `/rest/user/login`의 401 실패와 200 성공 응답을 구분해서 해석한다.
- bypass 성공 요청은 단순 로그인 실패보다 더 강하게 서술될 가능성이 있다.
- known_asset + `lab-*` 문맥에서는 내부 테스트 가능성을 함께 고려해야 한다.

### 기대 관찰 포인트
- prepare에서 200 성공 응답 2건이 후보로 포함되는지
- stage1에서 200 성공 2건과 401 실패 5건을 어떻게 다르게 분류하는지
- stage2에서 OpenAI와 Anthropic의 incident 강조 정도가 어떻게 갈리는지

## 6. 비교 메모

### OpenAI / Anthropic 차이
- OpenAI는 더 보수적이다.
- Anthropic은 200 성공 2건을 더 강하게 핵심 incident로 올린다.
- OpenAI는 401 실패 일부를 false positive 쪽으로 낮춘다.
- Anthropic은 401 실패까지 넓게 suspicious로 유지한다.

### 오탐 / 미탐 / 경계형
- **경계형**: 200 성공 2건
- **오탐 판단 차이**: 401 실패 5건 중 2건
- **공통 한계**: known_asset / `lab-*` 문맥 때문에 외부 공격 확정으로는 쓰기 어렵다

## 7. 최종 평가

이번 v2의 가장 중요한 변화는, 이전 라운드에서 빠졌던 **200 성공 bypass 2건이 prepare, stage1, stage2 전 단계에 반영되었다는 점**이다.

이로 인해 이번 비교는 단순 401 실패 반복 비교가 아니라, **성공한 인증 우회 정황까지 포함한 provider별 해석 차이 비교**가 되었다.

- OpenAI: 보수적, 운영 검토 중심
- Anthropic: 더 공격적, incident 승격과 severity 강화

다만 두 결과 모두 `192.168.56.110`이 known_asset이고 `lab-*` user-agent가 명시돼 있어, 현 시점에서는 **외부 침해 확정이 아니라 통제된 내부 실험 트래픽으로 보는 해석이 우선**이다.

### 후속 확인 필요 사항
- 200 성공 2건의 실제 토큰/세션 발급 여부 확인
- 해당 시각대 테스트 계획 대조
- 401 → 200 시계열 흐름 재구성
- `low_signal_fuzzing` 12건의 연계 여부 확인
