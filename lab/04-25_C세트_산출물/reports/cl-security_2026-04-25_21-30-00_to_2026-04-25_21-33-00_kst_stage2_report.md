## 8. 권고 조치
- **P1** 내부 보안 테스트 목적 및 타이밍 확인
  - 근거: 192.168.56.110이 known_asset 내부 IP이고 모든 사용자 에이전트가 명시적 테스트 라벨('lab-c-set-xss-*')을 포함하고 있습니다. 개발팀, 보안팀, QA팀에 확인하여 21:30~21:33 구간 예정된 테스트 활동이 있었는지, 도구가 무엇인지 파악해야 합니다.
- **P1** /rest/products/search 엔드포인트의 입력 처리 및 응답 로직 검토
  - 근거: 8건의 XSS 페이로드가 모두 동일 엔드포인트를 대상으로 하고 있으며, 일부는 200 application/json 응답으로 처리되었습니다. 응답 본문(30바이트)이 작아 현재로는 페이로드 반향 여부 확정이 어렵지만, 실제 응답 내용을 분석하여 입력값이 JSON에 포함되는지, HTML 컨텍스트로 렌더링되는지 점검 필요.
- **P2** XSS 페이로드 처리에 대한 WAF 또는 애플리케이션 필터 동작 확인
  - 근거: 다양한 인코딩 기법(URL, HTML 엔티티, 이중 인코딩, 혼합 대소문자)이 시도되었으나 일부는 200으로 처리되었습니다. 현재 적용된 XSS 방어 메커니즘(Content-Security-Policy, output encoding, input validation)이 이러한 페이로드를 충분히 차단/인코딩하는지 검토 필요.
- **P2** HTTP 500 오류 발생의 원인 분석
  - 근거: rank 1, 4, 5 incident에서 XSS 페이로드에 대해 500 text/html 응답이 반환되었습니다. 이는 애플리케이션 처리 실패를 의미하지만, 페이로드가 서버에 전달되고 처리 시도된 것을 확인해줍니다. 에러 로그와 애플리케이션 로그를 검토하여 500 원인이 무엇인지, 보안 관련인지 파악 필요.
- **P2** 네트워크 및 엔드포인트 모니터링을 통한 브라우저 XSS 실행 확인
  - 근거: Apache 로그만으로는 페이로드가 브라우저에서 실행되었는지 확정할 수 없습니다. 내부 테스트가 맞다면, 테스트 대상 클라이언트의 브라우저 개발자 도구 로그, 네트워크 추적(wireshark), 또는 example.invalid 수집 서버의 로그를 확인하여 실제 XSS 실행 여부 검증 필요.
- **P3** false_positive_review_candidates 검토: 교육용 SQL/XSS 검색 질의 모니터링
  - 근거: false_positive_review_candidates에 'educational_xss_keyword_search' 형태의 요청이 기록되어 있습니다. 현재는 rank 9로 평가된 자연어 검색이 이미 오탐 처리되었지만, 향후 유사한 보안 교육/문서 검색 질의를 필터링할 규칙 정제 필요.

## 9. 신뢰도와 한계
- 모든 출발지 IP(192.168.56.110)가 known_asset_ips에 포함되고 사용자 에이전트가 명시적 테스트 라벨을 포함하고 있어, 내부 자동화 테스트 또는 개발 환경 활동으로 평가하는 신뢰도는 높습니다. 다만 내부 IP 스푸핑 또는 악의적 내부자 가능성을 완전히 배제하지는 못합니다.
- Apache 웹 로그만으로는 XSS 페이로드의 브라우저 실행 성공을 확정할 수 없습니다. 200 status_code와 application/json resp_content_type은 서버가 요청을 처리했음을 의미하지만, 응답 본문이 30바이트로 작아 실제 페이로드 반향 여부를 Apache 로그에서 판단할 수 없습니다.
- 500 text/html 응답은 서버 처리 오류 정황을 나타내지만, 이를 XSS 실행 성공의 근거로 해석하지 않았습니다. 오류 메시지 자체가 XSS 취약할 수 있지만, 현재 응답 본문 분석 없이는 확정 불가능.
- document.cookie, document.location 등 브라우저 데이터 접근 시도가 명시적으로 탐지되었으나, 실제 쿠키 탈취, 외부 URL 리다이렉트 성공 여부는 별도 네트워크 모니터링(DNS, HTTP 아웃바운드) 또는 애플리케이션 로그 없이 확정할 수 없습니다.
- HTML 엔티티, URL 인코딩, 이중 인코딩 등 우회 기법이 탐지되었으나, 실제 우회 성공은 애플리케이션의 디코딩 순서와 검증 시점에 따라 달라집니다. Apache 로그만으로는 애플리케이션이 이를 복원하여 처리했는지 판단할 수 없습니다.
- 1건의 필터링된 요청(low_signal_fuzzing)이 존재하나, 현재 분석 범위에서 제외되었으므로 세부 내용 확인 불가. 이는 후보 밖 탐색성 요청으로 간주됨.

## 10. 발표용 한 줄 정리
분석 기간의 9건 요청은 모두 내부 자산 IP(192.168.56.110)에서 /rest/products/search 엔드포인트를 대상으로 한 의도적 XSS 테스트로 평가됩니다. 사용자 에이전트의 명시적 테스트 라벨('lab-c-set-xss-core-*', 'lab-c-set-xss-adv-*')과 집중된 시간대(21:30~21:33)를 고려하면 자동화된 내부 보안 검사 또는 개발 환경 점검일 가능성이 높습니다. 현재 외부 공격 위협은 낮으나, (1) 내부 테스트 목적 및 일정 확인, (2) 애플리케이션이 다양한 인코딩 기법의 XSS 페이로드를 어떻게 처리하는지 검증, (3) 실제 브라우저 XSS 실행 여부 확인 등이 필요합니다. 특히 rank 1 요청의 document.cookie 탈취 의도와 외부 URL 리다이렉트 시도는 고급 공격 시나리오이므로, 내부 테스트 목적이 확인되더라도 애플리케이션 방어 기능의 효과성을 검증하는 데 활용할 가치가 있습니다.
